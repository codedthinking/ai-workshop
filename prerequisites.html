Prerequisites

Install these before the workshop. If you get stuck, bring your laptop and we will help you at the start.

Required

A laptop with a terminal

All participants

macOS, Linux, or Windows with WSL. You need a working terminal where you can run shell commands. On macOS, the built-in Terminal.app or iTerm2 works. On Windows, install WSL 2 first.

Git

Version control

We will use git throughout the workshop. Install it if you do not have it already.

# macOS (comes with Xcode Command Line Tools) xcode-select --install # Ubuntu / Debian sudo apt install git # Check your installation git --version

An API key from a model provider

Hands-on

You need access to at least one LLM. We recommend setting up an OpenRouter account, which gives you access to many models through a single API key. Alternatively, get an API key from Anthropic (Claude) or OpenAI directly.

Budget about $5–10 of credit for the workshop day.

Install one coding agent

Pick at least one. Both are terminal-based AI coding agents. We will primarily demo with Claude Code, but OpenCode is a strong open-source alternative that works with any provider.

Claude Code

Recommended

Anthropic's agentic coding tool. Requires Node.js 18+ and an Anthropic API key (or a Claude Pro/Max subscription).

# Install via npm npm install -g @anthropic-ai/claude-code # Verify claude --version

Documentation: docs.anthropic.com/en/docs/claude-code

OpenCode

Open source

Open-source AI coding agent. Works with any LLM provider (OpenRouter, Anthropic, OpenAI, local models).

# Install curl -fsSL https://opencode.ai/install | bash # Verify opencode --version

Documentation: opencode.ai

Optional but useful

DuckDB

Act 3 exercises

We use DuckDB in the dodo demo and exercise. Install it if you want to follow along hands-on in Act 3.

# macOS brew install duckdb # Or download from # https://duckdb.org/docs/installation

Make

Likely pre-installed

GNU Make for running pipelines. Already installed on most macOS and Linux systems.

make --version

A project to work on

Bring your own

The exercises work best if you have a real research project — ideally a git repo with some data-wrangling scripts (.do, .py, .R, or .jl). If you do not have one ready, we provide a sample dataset.
